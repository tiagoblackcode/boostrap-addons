Add-ons for Bootstrap

Radio Group radio-group.js

About

Radio Group enables the radio and pseudo-radio input selection using bootstrap markup and hiding the usual form controls. It comes in two forms: in component mode, where inputs are [type="radio"] and live inside the element markup. Useful when making radio buttons out of dropdown lists. In its second form, it acts as a pseudo-radio input. This is useful when you have a .btn-group and you want to have a set of buttons to to change a hidden's input value.

Either forms support initial values but, in the pseudo-radio input form, the selected value and subsequent selections are grabbed from the button's value attribute.

Everytime an input changes, it still triggers the change event so that you can still handle additional workflows.

Attached to a button group

For the input to be properly changed you have to set id="some_id" and data-behaviour="radio-group" in the button group wrapper and data-target-for="some_id" in the hidden input you want to change using the button group. This won't break html validation as opposed to having input tags inside the button markup.

Markup

<div id="btn-group-radio" class="btn-group" data-behaviour="radio-group">
  <button type="button" class="btn" value="option1">Option 1</button>
  <button type="button" class="btn" value="option2">Option 2</button>
  <button type="button" class="btn" value="option3">Option 3</button>
</div>
<input type="hidden" name="btn-group" value="option1" data-target-for="btn-group-radio" />

Attached to a dropdown menu

When working with dropdown menus you can keep the input type="radio" markup inside the li tags with any nesting level. In this case, the component simply hides the input markup by providing a prettier interface.

Options
  • Option 1
  • Option 2
  • Option 3

Markup

<div class="dropdown btn-group">
  <a class="dropdown-toggle btn" data-toggle="dropdown">Options</a>
  <ul class="dropdown-menu" data-behaviour="radio-group">
    <li>
      <a href="#">Option 1</a>
      <input type="radio" name="" value="option1" checked />
    </li>
    <li>
      <a href="#">Option 2</a>
      <input type="radio" name="" value="option2" />
    </li>
    <li>
      <a href="#">Option 3</a>
      <input type="radio" name="" value="option3" />
    </li>
  </ul>
</div>

Confirm Popover confirm-popover.js

About

Confirm Popover plugins enables you to define actions that are only performed upon confirmation. The confirmation plugin uses the bootstrap-popover.js plugin to display a confirmation dialog to the user. It uses the same data- attributes as the popover plugin and also two additional attributes data-ok-text and data-cancel-text so it is possible for you to localize the dialog box.

The OK button inherits the attributes white-listed in the array given as options or data-attributes so it behaves like the original confirmable link, after the user confirms the action on the dialog. These attributes can be passed in as options or as data attributes and may contain whatever you like.

This plugin is inspired by Rails' data-confirm attribute, but presented in a nicer, cleaner way.

Example

Dangerous action

Markup

<a href="javascript:alert('Remote action was called')" 
   class="btn btn-danger" rel="confirm-popover" 
   data-content="Are you really really sure?" 
   data-ok-text="Yup!" data-cancel-text="Nope">
  Dangerous action
</a>

Usage

Like bootstrap-popover.js plugin, you hook the confirm-popover.js plugin using it's constructor, passing as argument the options you like. Similary, you can programaticaly show and hide the component, due to its inheritance from the bootstrap-popover.js 

$('[rel*=confirm-popover]').confirmPopover(options)
